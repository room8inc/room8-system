# Room8 セットアップガイド

## 🎯 はじめに

開発を始める前に、以下のサービスのアカウント作成が必要です。

**全て無料で作成できます！**

## 📋 必要なアカウント

### Phase 1（MVP開発）で必要なアカウント

1. **Vercel**（フロントエンド + バックエンド）
2. **Supabase**（データベース + 認証 + ストレージ）

4. **LINE Developers**（LINE連携・オプション）

## 🔧 アカウント作成手順

### 0. GitHub アカウント作成（推奨）

**URL**: https://github.com

**手順：**
1. GitHubの公式サイトにアクセス
2. 「Sign up」をクリック
3. Emailアドレス、パスワード、ユーザー名を入力
4. メールアドレス確認
5. アカウント作成完了

**プラン：**
- ✅ **無料プランで十分**（Public/Privateリポジトリ無制限）
- ✅ 月額料金なし

**重要：**
- Vercelと連携する場合、GitHubアカウントが必要
- コードのバージョン管理に必須
- 複数人で開発する場合も便利

**リポジトリ作成手順：**
1. GitHubにログイン
2. 「New repository」をクリック
3. リポジトリ名を入力: `room8-system`（任意）
4. **Private**を選択（推奨、個人情報を含む可能性があるため）
5. README.mdは既に作成済みなので不要
6. 「Create repository」をクリック

**リポジトリURL：**
- https://github.com/room8inc/room8-system.git

**次のステップ：**
- ローカルリポジトリとGitHubリポジトリを連携
- VercelとGitHubを連携（自動デプロイ設定）

### 1. Vercel アカウント作成

**URL**: https://vercel.com

**手順：**
1. Vercelの公式サイトにアクセス
2. 「Sign Up」または「Get Started」をクリック
3. **GitHubアカウントでサインアップ（推奨）**
   - GitHubアカウントと連携すると、自動デプロイが簡単
   - または、Email、GitLab、Bitbucketでも可能
4. GitHubの認証を許可
5. アカウント作成完了

**プラン：**
- ✅ **無料プランで開始可能**（Hobbyプラン）
- 月100GB転送、1000回ビルドまで無料
- 1日20〜30人の規模なら無料枠で十分

**料金：**
- **無料プラン**: 月額0円
- **Proプラン**: 月額$20（成長時にアップグレード）

**重要：**
- 開発中も無料で利用可能
- 本番環境でも無料枠で運用可能（1日20〜30人の規模なら）

### 2. Supabase アカウント作成

**URL**: https://supabase.com

**手順：**
1. Supabaseの公式サイトにアクセス
2. 「Start your project」をクリック
3. GitHubアカウントでサインアップ（推奨）
   - または、Emailでも可能
4. アカウント作成完了
5. プロジェクトを作成
   - プロジェクト名: `room8`（任意）
   - データベースパスワードを設定
   - リージョン: `Northeast Asia (Tokyo)`（日本に近いリージョン）

**プラン：**
- ✅ **無料プランで開始可能**（Freeプラン）
- 500MB データベース容量、1GB ファイルストレージまで無料
- 1日20〜30人の規模なら無料枠で十分

**料金：**
- **無料プラン**: 月額0円
- **Proプラン**: 月額$25（成長時にアップグレード）

**重要：**
- 開発中も無料で利用可能
- 本番環境でも無料枠で運用可能（1日20〜30人の規模なら）

### 3. Stripe アカウント作成

**URL**: https://stripe.com/jp

**手順：**
1. Stripeの公式サイトにアクセス
2. 「今すぐアカウントを作成」をクリック
3. Emailアドレス、パスワードを入力
4. メールアドレス確認
5. アカウント情報を入力（ビジネス情報など）
6. アカウント作成完了

**プラン：**
- ✅ **無料でアカウント作成可能**
- 決済手数料のみ（3.6% + 固定費）
- 月額料金なし

**料金：**
- **アカウント作成**: 無料
- **決済手数料**: 3.6% + 固定費（取引ごと）

**重要：**
- テストモードで開発可能（本番環境と同じ機能をテストできる）
- テストモードでは実際の決済は発生しない
- 本番環境に切り替える際に審査が必要

## 🔑 必要な情報（開発時に使用）

### Vercel
- **アカウント情報**: ログイン情報のみ
- **環境変数**: 後で設定（Supabase、StripeのAPIキーなど）

### Supabase
- **プロジェクトURL**: `https://xxxxx.supabase.co`
- **API Key（anon）**: Settings > API から取得
- **API Key（service_role）**: Settings > API から取得（サーバー側のみ）
- **データベースパスワード**: プロジェクト作成時に設定したパスワード

### Stripe
- **Publishable key**: 開発者 > API キー > 公開可能キー
- **Secret key**: 開発者 > API キー > シークレットキー
- **テスト用カード番号**: 
  - 成功: `4242 4242 4242 4242`
  - 失敗: `4000 0000 0000 0002`

## 🚀 次のステップ

アカウント作成完了後、以下を実行：

1. **GitHubリポジトリとローカルを連携**
   - ローカルリポジトリを初期化
   - GitHubリポジトリと連携
   - 最初のコミット＆プッシュ

2. **開発環境構築**
   - Node.jsのインストール
   - Next.jsプロジェクトの初期化
   - 必要なパッケージのインストール

3. **Supabase接続設定**
   - 環境変数にAPIキーを設定
   - データベースの初期セットアップ

4. **Vercelプロジェクト作成**
   - GitHubリポジトリと連携（自動デプロイ設定）
   - 環境変数を設定
   - 自動デプロイの確認

5. **Stripeテスト環境の設定**
   - 環境変数にAPIキーを設定
   - テストモードで動作確認

## 💡 注意事項

**重要：**
- APIキーは**絶対に公開しない**
- 環境変数で管理（Gitにコミットしない）
- `.env.local`ファイルを`.gitignore`に追加
